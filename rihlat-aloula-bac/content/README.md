# بنية المحتوى — رحلة الأولى باك

يفصل هذا التنظيم **المحتوى العلمي** عن **الواجهة البرمجية** (الميثاق، بند 9).

```
rihlat-aloula-bac/
├── content/                 # المحتوى المنظّم (مصدر الحقيقة)
│   ├── history/             # دروس التاريخ
│   │   └── lesson-XX-<slug>.json
│   └── geography/           # دروس الجغرافيا (لاحقًا)
├── scripts/
│   └── build_lessons.py     # يولّد صفحات HTML ثابتة من ملفات المحتوى
├── assets/css/lesson.css    # الهوية البصرية المعتمدة (منفصلة)
└── lessons/                 # الصفحات الثابتة المولّدة (لا تُحرّر يدويًا)
    └── <slug>.html
```

## المبادئ
- **المحتوى منفصل عن الواجهة**: يُحرَّر المحتوى في `content/*.json` فقط.
- **يعمل بلا JavaScript**: يُدمج النص كاملًا في HTML النهائي عند البناء، فلا اعتماد على
  Notion أو Drive وقت تصفح التلميذ (الميثاق، بند 10).
- **حقول الحالة** في كل ملف (`meta.status`, `meta.version`, `meta.source`) لتتبّع المراجعة والاعتماد.

## البناء
```bash
python3 scripts/build_lessons.py
```
يقرأ كل ملفات `content/**/*.json` ويولّد الصفحات في `lessons/`.

## حالة الدرس (meta.status)
`مصدر غير مفحوص` · `قيد المراجعة` · `في انتظار اعتماد الأستاذ` · `معتمد مضمونيًا` ·
`قيد الإدماج` · `قيد الاختبار` · `منشور` · `يحتاج إلى تصحيح`
